## 钱包类型概述

### 热钱包（Hot Wallet）

**定义**：持续连接互联网的钱包

```
常见形式：
- 手机App（MetaMask、Trust Wallet）
- 浏览器插件（MetaMask Extension）
- 桌面软件（Exodus、Electrum）
- 交易所账户（币安、OKX）

核心特征：
✓ 私钥存储在联网设备上
✓ 可随时快速交易
✓ 便于日常使用
✗ 面临网络攻击风险
```

### 冷钱包（Cold Wallet）

**定义**：完全离线的钱包

```
常见形式：
- 硬件钱包（Ledger、Trezor、OneKey）
- 纸钱包（打印的私钥/助记词）
- 钢板助记词（离线金属备份）
- 离线电脑（永不联网的专用设备）

核心特征：
✓ 私钥永不触网
✓ 极高的安全性
✓ 适合长期存储
✗ 交易需要额外步骤
```

## 详细功能对比

### 交易便利性

**热钱包**：
```
交易流程：
打开App → 输入金额 → 确认 → 完成（30秒内）

适用场景：
✓ 日常小额支付
✓ DeFi交互（Swap、Stake）
✓ NFT交易
✓ 频繁交易需求
```

**冷钱包**：
```
交易流程：
创建交易 → 导出到冷钱包 → 签名 → 广播（2-10分钟）

适用场景：
✓ 大额转账
✓ 长期存储
✓ 备份资产
✓ 低频交易
```

### 支持的资产和链

| 钱包类型 | 支持情况 | 举例 |
|----------|----------|------|
| **热钱包App** | 多链支持 | MetaMask支持ETH/BSC/Polygon等 |
| **硬件钱包** | 主流链支持 | Ledger支持5500+种资产 |
| **交易所** | 依赖平台 | 币安支持主流币种 |
| **专用钱包** | 单链优化 | Electrum专注BTC |

### 恢复能力

**热钱包恢复**：
```
方式一：助记词恢复
- 在新设备输入12/24个单词
- 资产立即恢复
- 风险：助记词泄露=资产被盗

方式二：云备份恢复
- 部分钱包支持iCloud/Google备份
- 便捷但增加攻击面
```

**冷钱包恢复**：
```
方式：助记词恢复
- 需要物理访问备份
- 可从任何兼容钱包恢复
- 需要妥善保管助记词

进阶方案：Shamir备份
- 将助记词分成多份
- 需要多份才能恢复
- 提高容错性
```

## 冷热钱包配合使用

### 最佳实践流程

#### 日常操作（小额）

```
热钱包准备：
1. 从冷钱包转入$1,000-5,000到热钱包
2. 用于Gas费、小额交易、DeFi交互
3. 余额低于$500时补充
4. 定期（每月）检查并归集到冷钱包

安全设置：
- 安装硬件钱包同款助记词（隔离小额）
- 或使用独立助记词（完全隔离）
- 开启所有安全选项
```

#### 大额操作

```
冷钱包操作：
1. 在热钱包（或电脑）创建交易
2. 导出交易到USB或二维码
3. 在冷钱包设备上签名
4. 将签名后的交易广播
5. 确认到账后记录

注意事项：
- 在安全环境下操作
- 确认收款地址正确（剪贴板劫持防护）
- 小额测试后再大额转账
```

### 安全转账检查清单

```
转账前：
□ 确认收款地址（首尾字符 + 中间几位）
□ 使用已验证的地址（地址簿）
□ 小额测试（$10-50）
□ 确认网络（ETH主网/Arbitrum/Polygon等）
□ 预估Gas费是否合理

转账中：
□ 在硬件钱包上确认地址
□ 确认金额正确
□ 确认网络正确
□ 保持设备连接稳定

转账后：
□ 在区块浏览器确认交易
□ 确认对方收到款项
□ 记录交易哈希
□ 更新账本记录
```

## 常见安全威胁与防护

### 热钱包主要威胁

#### 钓鱼攻击

```
攻击方式：
- 伪造MetaMask登录页面
- 发送钓鱼邮件诱导点击
- 社交媒体假客服私信

防护措施：
✓ 手动输入metamask.io
✓ 不点击任何钱包相关链接
✓ 验证官方域名
✓ 安装钓鱼防护插件
```

#### 恶意软件

```
攻击方式：
- 键盘记录器窃取密码
- 剪贴板劫持修改地址
- 屏幕录制窃取助记词

防护措施：
✓ 使用硬件钱包
✓ 定期杀毒扫描
✓ 不安装来路不明的软件
✓ 使用专用设备操作大额资产
```

#### 授权漏洞

```
攻击方式：
- 诱导授权无限额度
- 利用旧版合约漏洞
- 假空投诱导授权

防护措施：
✓ 使用 Revoke.cash 定期检查
✓ 不授权不信任的合约
✓ 使用 Pocket Universe 预览交易
✓ 大额资产不放热钱包
```

### 冷钱包主要威胁

#### 物理安全

```
风险场景：
- 硬件钱包丢失/被盗
- 助记词被拍照/复印
- 家庭火灾/水灾损坏

防护措施：
✓ 使用金属助记词板
✓ 分散存储（家中+银行保险箱+可信亲友）
✓ 使用Shamir分片备份
✓ 定期检查备份完整性
```

#### 供应链攻击

```
攻击方式：
- 购买二手/翻新设备
- 设备被预先植入恶意程序
- 包装被拆封后重新封装

防护措施：
✓ 官方渠道购买
✓ 检查包装完整性
✓ 首次使用初始化（生成新助记词）
✓ 验证设备固件签名
```

#### 社会工程

```
攻击方式：
- 假冒客服索要助记词
- "技术支持"远程协助
- 钓鱼电话诱导操作

防护措施：
✓ 永不透露助记词
✓ 官方客服不会索要私钥
✓ 不接受远程协助
✓ 建立固定安全流程
```
