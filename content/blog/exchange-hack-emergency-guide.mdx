## 发现被盗的第一反应

### 保持冷静，快速确认

```
确认清单（1分钟内完成）：
□ 是否有未经授权的交易通知？
□ 登录时是否有异常提示？
□ 账户余额是否异常减少？
□ 是否有陌生设备登录记录？
□ 是否收到密码/2FA变更通知？

重要提醒：
- 不要立即修改密码（可能触发更多限制）
- 不要先通知攻击者（可能加速资产转移）
- 不要慌张操作（避免误操作）
```

### 快速评估损失范围

```
检查项目：
1. 现货账户余额
2. 合约/杠杆仓位
3. 理财/质押资产
4. API Key是否被滥用
5. 提现记录（资金去向）

记录信息：
- 最后正常操作时间
- 发现异常的时间
- 异常交易的时间
- 涉及的币种和金额
```

## 黄金30分钟应急流程

### 0-5分钟：立即止损

#### 步骤1：冻结账户

```
Binance冻结：
1. 访问 binance.com（确保是官方网站）
2. 点击右下角客服图标
3. 输入 "Account hacked emergency"
4. 选择 "Freeze my account"
5. 按提示完成身份验证
6. 账户将被立即冻结

OKX冻结：
1. 访问 okx.com
2. 点击右下角客服
3. 输入 "账户被盗紧急冻结"
4. 选择账户冻结选项
5. 完成身份验证
6. 账户冻结

其他交易所：
- 查找"安全中心"或"帮助中心"
- 搜索"冻结账户"或"紧急冻结"
- 或通过官方客服电话/邮件
```

#### 步骤2：撤销所有API Key

```
操作路径（以Binance为例）：
1. 登录账户（如果可以）
2. 进入 API Management
3. 删除所有API Key
4. 禁用所有第三方应用授权
5. 检查IP白名单

如果无法登录：
- 联系客服要求撤销所有API
- 提供身份验证信息
```

### 5-15分钟：通知交易所

#### 提交盗币申诉

```
必备信息：
1. 账户基本信息
   - 注册邮箱/手机号
   - UID/User ID
   - 注册时间

2. 事件信息
   - 发现被盗时间
   - 异常交易记录（截图）

3. 设备信息
   - 常用设备类型
   - 常用登录地点
   - 是否有陌生设备登录

4. 联系信息
   - 紧急联系电话
   - 备用邮箱
   - 可接收短信的号码
```

#### 链上追踪

```
使用工具追踪资金：
1. Etherscan（ETH及ERC20）
```

```
- 明确说明是"网络盗窃"
- 提供完整的证据链

注意事项：
```

```
✓ 有明确的资金流向
✓ 交易所配合度高

中等可能性（30-50%）：
△ 资金已提现到链上
△ 延迟冻结（1-2小时内）

低可能性（10-20%）：
✗ 资金已通过混币器
```

## 防范于未然：安全加固

### 账户安全最佳实践

#### 强密码策略

```
密码要求：
✓ 至少16位字符
✓ 大小写字母 + 数字 + 符号
✓ 不使用任何字典词汇
✓ 不与任何其他账户共享
✓ 每3个月更换一次

密码管理器推荐：
- 1Password
- Bitwarden
- LastPass（不推荐，有泄露史）
```

#### 2FA配置

```
2FA优先级：
1. YubiKey/硬件安全密钥（最安全）
2. Google Authenticator（推荐）
3. Authy（有云端备份）
4. 短信验证（最不安全，但比没有好）

⚠️ 切勿使用：
- 邮箱作为2FA
- 基于SIM卡的验证
```

#### 登录保护

```
必开功能：
□ 登录邮箱通知
□ 异地登录提醒
□ 新设备确认
□ 提现邮箱确认
□ 提现2FA确认

白名单设置：
□ 提现地址白名单
□ 登录IP白名单
□ API IP白名单
```

### 日常安全检查清单

```
每日检查：
□ 账户登录记录
□ 未读安全通知
□ 资产变动提醒

每周检查：
□ API Key活动
□ 授权应用列表
□ 登录设备管理

每月检查：
□ 密码更新
□ 2FA备份码检查
□ 安全设置审查
□ 提现地址验证
```
