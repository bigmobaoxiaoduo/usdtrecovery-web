## What is a Rug Pull (Exit Scam)?

### Definition

A Rug Pull refers to project teams suddenly withdrawing funds from liquidity pools after attracting investor capital, causing token prices to crash to zero.

### Main Types

| Type | Description | Percentage |
|------|-------------|------------|
| **Hard rug** | Directly withdraw liquidity | 45% |
| **Soft rug** | Team stops development, slowly goes to zero | 30% |
| **Smart contract exploits** | Reserve backdoors to steal funds | 20% |
| **Market manipulation** | Whales pump and dump | 5% |

### Common Exit Tactics

#### 1. Liquidity Withdrawal (Most Common)

```
Operation method:
1. Project team provides initial liquidity (e.g., 100 ETH + tokens)
2. Attract investors to buy tokens
3. Token price rises, liquidity increases
4. Project team suddenly withdraws all liquidity
5. Token price crashes to zero, investors cannot sell
```

#### 2. Smart Contract Backdoors

```
Common backdoors:
- Hidden owner permissions (can mint unlimited tokens)
- Whitelist function (specific addresses can trade)
- Transaction tax modification (can set 100% transaction tax)
- Blacklist function (prevent specific addresses from selling)
```

#### 3. Flash Loan Attacks

```
Attack flow:
1. Use flash loans to borrow large amounts of funds
2. Manipulate price oracles
3. Arbitrage from protocol
4. Repay flash loan
5. Leave emptied protocol
```

## Due Diligence Checklist

### Step 1: Team Background Investigation

#### Basic Information Verification

```
Must-check list:
□ Real names of team members
□ LinkedIn profile verification
□ Past project experience
□ GitHub technical contributions
□ Social media activity
```

**Practical tools**:
- LinkedIn: Verify professional background
- GitHub: Verify technical capability
- Twitter: View historical statements
- Wayback Machine: View historical web pages

#### Anonymous Team Risk Assessment

```
Anonymous ≠ necessarily scam, but requires more caution

If team is anonymous, additionally check:
□ Whether code quality is professional
□ Whether there are well-known audits
□ Whether community has technical expert endorsements
□ How long project has been running
□ Whether funds are secure (multi-sig, timelock)
```

### Step 2: Smart Contract Audit

#### Audit Report Check

```
Well-known audit companies:
- CertiK
- OpenZeppelin
- Trail of Bits
- ConsenSys Diligence
- PeckShield

Check points:
□ Whether it's a well-known audit company
□ Whether audit date is recent
□ Whether serious issues found are fixed
□ Whether there are unfixed high-risk vulnerabilities
```

#### Self Code Check

```
Use tools:
- Etherscan: View contract code
- DeFi Safety: Security score
- RugDoc: Risk check

Check list:
□ Whether contract is verified
□ Whether there is owner permission
□ Whether there is mint function
□ Whether there is blacklist function
□ Whether transaction tax can be modified
```

### Step 3: Token Economic Model Analysis

#### Holdings Distribution Check

```
Use tools:
- Etherscan: Token Holders
- DeBank: Token analysis
- Bubblemaps: Visualize holdings distribution

Danger signals:
□ Top 10 addresses hold >50% (unless exchanges)
□ Team wallet not locked
□ Large amount of tokens concentrated in contract address
□ Recent large token transfers
```

#### Liquidity Analysis

```
Key indicators:
1. Liquidity/Market Cap Ratio
   - Healthy: >20%
   - Warning: 10-20%
   - Danger: <10%

2. Liquidity Lock
   - Must check if locked
   - Lock period at least 6-12 months
   - Use reliable platforms like Uncx or Team Finance

3. LP Token Ownership
   - Who holds LP tokens
   - Whether liquidity can be withdrawn anytime
```

### Step 4: Community and Marketing Assessment

#### Community Health

```
Check channels:
□ Discord activity
□ Telegram group quality
□ Twitter real engagement
□ Reddit discussion atmosphere

Red flags:
- Bot-filled
- No questioning allowed
- Only price discussions, no technical discussions
- Admins frequently mute users
```

#### Marketing Strategy Assessment

```
Healthy project marketing:
✓ Technology-oriented content
✓ Transparent development progress
✓ Community-driven growth
✓ Long-term roadmap

Danger signals:
✗ Excessive return promises
✗ Celebrity endorsements (paid promotion)
✗ Recruitment rewards
✗ "Last chance" urgency
```

**Red flags**:
- ❌ Website had many spelling errors

**Outcome**:

- Token went to zero
- Countless investors lost everything

**Lesson**:
> If you can't sell tokens, it's a honeypot!

## Risk Management Strategies

### Investment Portfolio Allocation

```
DeFi investment principles:
- Only invest funds you can afford to lose
- Single project not exceeding 5% of total position
- New project investment not exceeding 1% of total position

Recommended allocation:
- 70%: Mainstream currencies (BTC, ETH)
- 20%: Mainstream DeFi protocols (Aave, Uniswap, etc.)
- 10%: High-risk new projects (further diversified)
```

### Take Profit and Stop Loss Strategy

```
Set rules:
1. Stop loss
   - Single project loss >50%: Reassess
   - Loss >80%: Consider stop loss

2. Take profit
   - Profit >2x: Recover principal
   - Profit >5x: Take half profit
   - Remaining portion hold long-term

3. Rebalancing
   - Evaluate monthly
   - Appropriately reduce positions in profitable projects
   - Supplement to more stable assets
```

### Continuous Monitoring Checklist

```
Post-investment continuous monitoring:

Weekly checks:
□ Price anomalies (>30% fluctuation)
□ Liquidity changes
□ Social media updates

Monthly checks:
□ GitHub code updates
□ Team dynamics
□ Competitor developments
□ Regulatory updates

Emergency exit signals:
🚨 Liquidity suddenly withdrawn
🚨 Core team member resigns
🚨 Contract permission changes
🚨 Large token transfers
```

## If You've Already Invested in Suspicious Projects

### Emergency Assessment

```
Immediately check:
1. Can you sell tokens?
   - Test small amount sale
   - If can't sell, may be honeypot

2. Liquidity status
   - Check if liquidity still exists
   - Check if LP is locked

3. Team dynamics
   - Whether social media is active
   - Whether there are abnormal statements

4. Community atmosphere
   - Whether questioning is allowed
   - Whether anyone reports problems
```

### Exit Strategy

```
Scenario 1: Can trade normally
- If already profitable: Consider taking profit in batches
- If at loss: Assess whether worth continuing to hold

Scenario 2: Suspect imminent exit
- Sell immediately (even at loss)
- Don't wait to recover losses
- Stop loss in time to preserve principal

Scenario 3: Already exited
- Save all evidence
- Record transaction hashes
- Contact other victims
- Report to police and seek professional help
```
