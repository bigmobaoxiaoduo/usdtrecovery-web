## First Reaction Upon Discovery

### Stay Calm, Quickly Confirm

```
Confirmation Checklist (Complete within 1 minute):
□ Are there unauthorized transaction notifications?
□ Are there abnormal prompts during login?
□ Is account balance abnormally reduced?
□ Are there unfamiliar device login records?
□ Did you receive password/2FA change notifications?

Important Reminders:
- Don't immediately change password (may trigger more restrictions)
- Don't notify the attacker first (may accelerate asset transfer)
- Don't panic operate (avoid mistakes)
```

### Quickly Assess Loss Scope

```
Check Items:
1. Spot account balance
2. Contract/leverage positions
3. Earn/staking assets
4. Whether API Key is abused
5. Withdrawal records (fund destination)

Record Information:
- Last normal operation time
- Time when anomaly was discovered
- Time of abnormal transactions
- Involved currencies and amounts
```

## Golden 30-Minute Emergency Process

### 0-5 Minutes: Immediate Stop Loss

#### Step 1: Freeze Account

```
Binance Freeze:
1. Visit binance.com (ensure it's the official website)
2. Click customer service icon in bottom right corner
3. Enter "Account hacked emergency"
4. Select "Freeze my account"
5. Complete identity verification as prompted
6. Account will be frozen immediately

OKX Freeze:
1. Visit okx.com
2. Click customer service in bottom right corner
3. Enter "account hacked"
4. Select account freeze option
5. Complete identity verification
6. Account frozen

Other Exchanges:
- Find "Security Center" or "Help Center"
- Search for "freeze account" or "emergency freeze"
- Or through official customer service phone/email
```

#### Step 2: Revoke All API Keys

```
Operation Path (Binance example):
1. Login to account (if possible)
2. Enter API Management
3. Delete all API Keys
4. Disable all third-party app authorizations
5. Check IP whitelist

If unable to login:
- Contact customer service to revoke all APIs
- Provide identity verification information
```

### 5-15 Minutes: Notify Exchange

#### Submit Theft Appeal

```
Required Information:
1. Account Basic Information
   - Registered email/phone number
   - UID/User ID
   - Registration time

2. Incident Information
   - Time when hack was discovered
   - Abnormal transaction records (screenshots)
   - Fund destination (if known)

3. Device Information
   - Regularly used device types
   - Regular login locations
   - Whether there were unfamiliar device logins

4. Contact Information
   - Emergency contact phone
   - Backup email
   - Number that can receive SMS
```

### 15-30 Minutes: Tracking and Reporting

#### On-Chain Tracking

```
Use Tools to Track Funds:
1. Etherscan (ETH and ERC20)
   - Enter hacked address
   - View latest transactions
   - Track fund destination

2. BscScan (BSC Chain)
3. Tronscan (TRC20)
4. Arkham (address tags and analysis)
5. Chainalysis (professional tracking)

Record Information:
- Hacker wallet address
- Fund transfer path
- Whether it flowed into exchanges
- Transaction hashes (TxHash)
```

#### Police Report

```
Police Report Materials:
1. Identity proof
2. Exchange account information
3. Proof of stolen amount
4. Transaction record screenshots
5. Hacker address information
6. Timeline organization

Key Points for Reporting:
- Clearly state it's "cyber theft"
- Provide complete evidence chain
- Explain fund destination (if known)
- Request case filing receipt

Note:
- Some regions have limited experience with crypto cases
- May require multiple communications
- Stay patient and follow up continuously
```

## Post-Theft Asset Recovery

### Recovery Possibility Assessment

```
High Probability Recovery (60-80%):
✓ Funds still inside exchange (not withdrawn)
✓ Timely freeze (within 30 minutes)
✓ Clear fund flow
✓ High exchange cooperation

Medium Probability (30-50%):
△ Funds already withdrawn to chain
△ Delayed freeze (within 1-2 hours)
△ Complex fund flow

Low Probability (10-20%):
✗ Funds already through mixer
✗ Delayed processing (>24 hours)
✗ Flowed to decentralized platforms
```

## Prevention: Security Hardening

### Account Security Best Practices

#### Strong Password Strategy

```
Password Requirements:
✓ At least 16 characters
✓ Uppercase + lowercase + numbers + symbols
✓ No dictionary words
✓ Not shared with any other accounts
✓ Change every 3 months

Password Manager Recommendations:
- 1Password
- Bitwarden
- LastPass (Not recommended, has breach history)
```

#### 2FA Configuration

```
2FA Priority:
1. YubiKey/Hardware Security Key (Most secure)
2. Google Authenticator (Recommended)
3. Authy (has cloud backup)
4. SMS verification (Least secure, but better than nothing)

⚠️ Never use:
- Email as 2FA
- SIM-based verification
```

#### Login Protection

```
Must-Enable Features:
□ Login email notification
□ Abnormal login reminder
□ New device confirmation
□ Withdrawal email confirmation
□ Withdrawal 2FA confirmation

Whitelist Settings:
□ Withdrawal address whitelist
□ Login IP whitelist
□ API IP whitelist
```

### Daily Security Checklist

```
Daily Checks:
□ Account login records
□ Unread security notifications
□ Asset change alerts

Weekly Checks:
□ API Key activity
□ Authorized app list
□ Login device management

Monthly Checks:
□ Password update
□ 2FA backup code check
□ Security settings review
□ Withdrawal address verification
```
