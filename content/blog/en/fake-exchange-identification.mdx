## Common Types of Fake Exchanges

### Type 1: Completely Fake Counterfeit Exchanges

**Characteristics**:
- No real operations whatsoever
- Crudely made websites
- Cannot withdraw after deposit
- Customer service disappears or gives perfunctory responses

**Typical tactics**:
1. Heavy promotion on social media
2. Promise high returns (1-5% daily returns)
3. Induce users to deposit
4. Prevent withdrawals for various reasons
5. Eventually run away

### Type 2: Phishing Exchanges

**Characteristics**:
- Mimic interfaces of well-known exchanges
- Domains similar to official ones
- Induce input of account passwords
- Steal user assets

**Typical tactics**:
1. Send phishing links (email, SMS, social software)
2. User logs in and assets are stolen
3. Or induce downloading fake apps
4. Steal seed phrases and transfer assets

### Type 3: Ponzi Scheme Exchanges

**Characteristics**:
- Normal trading allowed initially
- Later restrict withdrawals for various reasons
- Induce recruitment (pyramid scheme model)
- Eventually run away

**Typical tactics**:
1. Normal operations to build trust
2. Launch "high-yield wealth management" products
3. Require locked funds
4. Suspend withdrawals for "system upgrades"
5. Run away

### Specific Identification Techniques

#### 1. Domain Verification

**Danger signals**:
```
Official: binance.com
Phishing: binance.co, binancee.com, bińance.com

Official: okx.com
Phishing: okex.com (old domain abandoned), oxk.com

Official: gate.io
Phishing: gate-io.com, gatte.io
```

**Verification methods**:
1. Find official domain through search engines
2. Compare every character
3. Watch for Unicode characters (e.g., ń vs n)
4. Save correct domain in bookmarks

#### 2. SSL Certificate Check

```
Steps:
1. Click lock icon in address bar
2. View certificate details
3. Confirm issuing authority
4. Confirm domain match
```

#### 3. Company Information Verification

**Must-check information**:
- Company registration information
- Office address (whether it actually exists)
- Team members (whether verifiable)
- Compliance licenses (MSB, VASP, etc.)

**Query tools**:
- OpenCorporates (international companies)
- Local business registries
- Regulatory websites

#### 4. Social Media Verification

**Check points**:
```
Twitter/X:
- Whether follower count matches engagement
- Account verification status
- Quality of historical tweets

Telegram:
- Official group member count
- Administrator identities
- Whether sensitive topics are banned

Reddit:
- Community activity
- Authentic user reviews
```

## Phishing Website Identification Techniques

### Common Phishing Methods

#### Method 1: Link Deception

```
Real link:
https://www.binance.com/en/trade/BTC_USDT

Phishing links:
https://www.binance.com-wallet.info/login
https://binance.com.signin.verify-account.net
https://binance-com-login.com
```

**Identification points**:
- Whether domain main body is correct
- Watch for additional subdomains
- Check if path is abnormal

#### Method 2: Email Spoofing

**Danger signals**:
- Sender address has subtle differences
- Urgent wording ("account will be frozen")
- Require clicking links to verify
- Grammar errors or strange expressions

**Verification methods**:
```
Don't click links in emails!
Correct approach:
1. Directly type official website to visit
2. Log in to check if relevant notifications exist
3. Or contact official customer service to confirm
```

#### Method 3: QR Code Phishing

```
Dangerous scenarios:
- "Recharge QR codes" shared in groups
- "Receiving codes" in social software
- Claiming "scan to receive airdrop"

Prevention methods:
- Only scan QR codes from official channels
- Verify address is correct after scanning
- Test small amount before large transfer
```

### Technical Identification Methods

#### 1. Webpage Source Code Check

```
Right click → View page source
Check:
- Whether suspicious external scripts are loaded
- Whether form submission URL is correct
- Whether keyboard logging code exists
```

#### 2. Network Request Monitoring

```
Use browser developer tools:
F12 → Network → Observe requests

Watch for:
- Data sent to unknown servers
- Abnormal API calls
- Encrypted suspicious communications
```

#### 3. Third-Party Tools

```
Recommended tools:
- VirusTotal: Detect malicious websites
- URLVoid: Website reputation check
- PhishTank: Phishing website database
```

## Typical Case Studies

### Case 1: "BitXPro" Fake Exchange Exit Scam

**Time**: June 2024

**Tactics**:
1. Promoted in Telegram groups, claiming "Singapore's top exchange"
2. Offered "principal-guaranteed high-yield wealth management", 2% daily returns
3. Allowed small withdrawals initially to build trust
4. Induced users to invest large amounts
5. Suspended withdrawals for "system upgrades"
6. Website inaccessible after one week, customer service disappeared

**Victim count**: Approximately 3,000 people
**Loss amount**: Approximately $80 million

**Identification points**:
- ❌ Could not find Singapore registration information
- ❌ Rough website design, heavy template traces
- ❌ Promised fixed returns (2% daily)
- ❌ Required recruitment

### Case 2: Phishing Website Stealing Coinbase Accounts

**Time**: September 2024

**Tactics**:
1. Send phishing emails claiming "account abnormal"
2. Links point to coinbase-verify.com (fake website)
3. Interface almost identical to official
4. Induce users to enter account, password, 2FA code
5. Immediately log in to real account and transfer assets

**Identification points**:
- ❌ Wrong domain (not coinbase.com)
- ❌ Abnormal email sender address
- ❌ Urgent wording creating panic
- ✅ Should directly visit official website instead of clicking links

### Case 3: Fake Customer Service Scam

**Time**: November 2024

**Tactics**:
1. Search for "Binance" related tweets on Twitter
2. Private message claiming "I'm Binance customer service"
3. Claim "detected abnormality in your account"
4. Induce downloading "security verification app"
5. App is actually Trojan, steals seed phrase

**Identification points**:
- ❌ Legitimate exchanges don't proactively private message
- ❌ Won't require downloading additional software
- ❌ Won't ask for seed phrase or private key

## Prevention Measures: Protect Your Assets

### Basic Protection

#### 1. Establish Secure Access Habits

```
✓ Use bookmarks to access exchanges
✓ Manually type URLs, don't click links
✓ Check domain three times before logging in
✓ Enable two-factor authentication (2FA)
✓ Use dedicated email for registration
```

#### 2. Device Security

```
✓ Install antivirus software and keep updated
✓ Don't use jailbroken/rooted devices
✓ Regularly check browser plugins
✓ Use password manager to generate strong passwords
```

#### 3. Information Protection

```
✗ Don't publicly disclose holdings on social media
✗ Don't share account screenshots (even with blur)
✗ Don't use public WiFi to log in to exchanges
✗ Don't log in to accounts on unfamiliar devices
```

### Advanced Protection

#### 1. Use Hardware Security Keys

```
Recommended products:
- YubiKey 5 Series
- Google Titan Key
- Thetis FIDO2 Key

Advantages:
- Physical key, cannot be stolen remotely
- Prevent phishing attacks
- Even if password leaked, cannot log in
```

#### 2. Whitelist Settings

```
Most exchanges support:
✓ Withdrawal address whitelist
✓ IP whitelist
✓ Device whitelist

After setting:
- Can only withdraw to specified addresses
- Can only log in from whitelisted IPs
- New devices require additional verification
```

#### 3. Asset Diversification

```
Don't put all assets in one exchange:
- Main assets: Cold wallet storage
- Trading funds: Top-tier exchanges
- Small test amounts: Secondary exchanges

Recommended allocation:
- 70%: Cold wallet (Ledger/Trezor)
- 20%: Top-tier exchanges (Binance/OKX)
- 10%: Hot wallet/daily trading
```

## What to Do After Discovering Suspicious Exchanges

### Immediate Action

```
If you've visited a suspicious website:

1. Exit immediately and clear browser data
   - Clear cookies and cache
   - Revoke any authorizations

2. Change relevant passwords
   - If you entered password, change immediately
   - Enable 2FA (if not already enabled)

3. Check account security
   - View login history
   - Check API keys
   - Confirm asset security
```

### If You've Been Scammed

```
Step 1: Stop loss
- Immediately transfer remaining assets to new wallet
- Revoke all suspicious authorizations
- Save all evidence

Step 2: Report to police
- Collect all transaction records
- Save website screenshots
- Report to local public security organs

Step 3: Seek help
- Contact professional recovery team
- Expose on social media (to help others)
- Report to relevant platforms
```
