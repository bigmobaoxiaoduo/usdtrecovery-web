## Basic Understanding of Wallet Security

### What is a Cryptocurrency Wallet?

A cryptocurrency wallet is not a place to "store" coins, but a **tool to manage private keys**.

- **Private key**: The only credential controlling assets
- **Public key/Address**: The "account" for receiving assets
- **Seed phrase**: Human-readable form of the private key

**Core Principle**:
> Whoever holds the private key owns the assets. Wallet service providers don't store your assets, only provide management tools.

### Three Root Causes of Security Incidents

| Rank | Cause | Percentage | Typical Scenario |
|------|-------|------------|------------------|
| 1 | Seed phrase leak | 35% | Screenshots on phone, sent to "customer service", stored in cloud |
| 2 | Malicious contract authorization | 28% | Clicked phishing links, authorized malicious contracts |
| 3 | Fake wallet app | 20% | Downloaded counterfeit wallet, imported seed phrase |
| 4 | Device compromise | 12% | Phone/computer infected with trojan |
| 5 | Others | 5% | Social engineering, brute force, etc. |

## Cold Wallet vs Hot Wallet: How to Choose?

### Core Differences Between the Two Wallet Types

| Feature | Cold Wallet (Hardware Wallet) | Hot Wallet (Software Wallet) |
|---------|------------------------------|------------------------------|
| **Private key storage** | Offline device | Internet-connected device |
| **Security** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **Convenience** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| **Use case** | Long-term large asset storage | Daily small transactions |
| **Price** | $50-300 | Free |
| **Representative products** | Ledger, Trezor, OneKey | MetaMask, Trust Wallet |

### Selection Recommendations

#### Scenarios for Using Cold Wallets

✅ **Strongly recommended for cold wallets**:
- Holding crypto assets worth > $10,000
- Long-term holding with infrequent trading
- Corporate or institutional asset management
- Extremely high security requirements

#### Scenarios for Using Hot Wallets

✅ **Acceptable to use hot wallets**:

**Hot wallet usage principles**:
- Only store amounts you "can afford to lose"

## Seed Phrase Security: Your Digital Lifeline

### What is a Seed Phrase?

A seed phrase (Recovery Phrase / Seed Phrase) is the **seed for generating private keys**, usually consisting of 12 or 24 English words.

**Key Understanding**:
- Anyone with the seed phrase = complete control of the wallet
- Seed phrases cannot be reset or recovered
- Seed phrase leak = inevitable asset theft

### DOs and DON'Ts for Seed Phrase Storage

#### ❌ Absolutely Prohibited Practices

| Dangerous Behavior | Risk Level | Real Case |
|-------------------|------------|-----------|
| Screenshot saved in phone gallery | 🔴 Extremely high | Phone hacked, lost 30 ETH |
| Stored in cloud services | 🔴 Extremely high | iCloud stolen, lost 100,000 USDT |
| Sent through social software | 🔴 Extremely high | Sent to "customer service", instantly stolen |
| Saved in computer documents | 🟠 High | Computer infected with trojan, seed phrase stolen |
| Photographed on phone | 🟠 High | Phone lost, assets stolen |
| Told to anyone | 🔴 Extremely high | Acquaintance crime, unrecoverable |

#### ✅ Recommended Secure Practices

**Method 1: Paper Backup (Basic Plan)**

```
Steps:
1. Handwrite seed phrase on paper (at least 2 copies)
2. Check 3 times to ensure accuracy
3. Store in fireproof and waterproof safe location
4. Consider bank safe deposit box
```

**Method 2: Metal Seed Plate (Recommended Plan)**

- Fire resistant (up to 1000°C+)
- Waterproof and corrosion resistant
- Physical damage resistant
- Suitable for long-term storage

**Recommended products**:
- Cryptosteel Capsule ($80)
- Billfodl ($40)
- Metal seed plates ($15-30)

**Method 3: Sharded Storage (Advanced Plan)**

Split 24 words into multiple groups and store separately:

```
Example (2-of-3 scheme):
- Location A: Words 1-8
- Location B: Words 9-16  
- Location C: Words 17-24

Security:
- Any two locations can recover wallet
- Single location leak won't cause asset loss
- Even if one location is destroyed, wallet can still be recovered
```

### Seed Phrase Verification Checklist

After backing up seed phrase, complete these checks:

- [ ] Handwritten text is clear and legible
- [ ] Checked 3 times, confirmed correct
- [ ] Numbered in order (1-12 or 1-24)
- [ ] Tested wallet recovery (using backup device)
- [ ] Stored in secure location
- [ ] Family or trusted person knows storage location (optional)

## Hot Wallet Security Configuration Guide

### MetaMask Security Configuration

#### Basic Settings

```
1. Set strong password
   - At least 12 characters
   - Include uppercase, lowercase, numbers, symbols
   - Don't use common words

2. Enable biometric (mobile)
   - Fingerprint or facial recognition
   - Add extra layer of protection

3. Auto-lock settings
   - Set 1-5 minute auto-lock
   - Prevent theft after leaving
```

#### Advanced Security Settings

**1. Hide token amounts**
```
Settings path:
Settings → Security & Privacy → Show balance and token price ON/OFF
```
- Prevent shoulder surfing of asset information

**2. Disable ENS auto-resolution**
```
Settings → Privacy → ENS domain lookup → OFF
```
- Prevent accessing malicious ENS domains

**3. Turn off unnecessary APIs**
```
Settings → Security & Privacy →
- Phishing detection: ON
- Participate in MetaMetrics: OFF (optional)
```

### Trust Wallet Security Configuration

```
Recommended settings:
✓ Enable app lock (PIN or biometric)
✓ Enable transaction confirmation (requires password/fingerprint)
✓ Disable DApp browser (if not needed)
✓ Regularly check authorized DApps
```

## Authorization Management: The Most Overlooked Risk

### What is Token Authorization?

When you use DeFi protocols (like Uniswap, Aave), you need to "authorize" contracts to access your tokens. This authorization:
- Usually has no upper limit (allows transferring entire balance)
- Remains valid long-term (unless actively revoked)
- Can be exploited by malicious contracts to steal assets

### Real Case: Authorization Theft

**Case**: User participated in some "airdrop" activity, authorized malicious contract, 500,000 USDT was transferred within 1 hour.

**Reason**: Malicious contract exploited unlimited authorization, could transfer user assets at any time after obtaining authorization.

### Authorization Management Best Practices

#### 1. Regularly Check Authorizations

**Recommended tools**:
- **Revoke.cash** (https://revoke.cash) - Most popular
- **Unrekt** (https://app.unrekt.net) - Multi-chain support
- **DeBank** (https://debank.com) - Comprehensive asset management

**Check frequency**:
- Heavy DeFi users: Weekly
- Regular users: Monthly
- Check immediately if abnormalities discovered

#### 2. Use Limited Authorization

Some wallets support "limited authorization":
```
MetaMask operation:
1. Click "Edit permission" when authorizing
2. Select "Custom spend limit"
3. Set specific amount (e.g., 1000 USDT)
4. Confirm authorization
```

#### 3. Promptly Revoke Unused Authorizations

```
Steps to revoke using Revoke.cash:
1. Visit https://revoke.cash
2. Connect wallet (read-only mode is safer)
3. View all authorization list
4. Click "Revoke" for unused authorizations
5. Confirm transaction in wallet (small gas fees apply)
```

### Authorization Security Checklist

- [ ] Check all authorizations monthly
- [ ] Revoke unused authorizations
- [ ] Use limited authorization for new projects
- [ ] Don't authorize unknown contracts
- [ ] Regularly check authorization list

## Preventing Fake Wallets and Phishing Attacks

### Fake Wallet App Identification

#### Common Tactics

| Tactic | Characteristics | Prevention Method |
|--------|----------------|-------------------|
| High-fake websites | Slight domain differences | Carefully check domain, use bookmarks |
| Counterfeit apps | Third-party app store piracy | Download only from official website |
| Fake customer service | Proactive private messages offering "help" | Official never proactively private messages |
| Airdrop诱导 | Download app to receive airdrop | Beware of free lunches |

#### Secure Download Checklist

Before downloading wallet app, check:

- [ ] Official website domain correct (using https)
- [ ] Download from official website link
- [ ] Verify app developer information
- [ ] Check download volume and ratings
- [ ] Read user reviews
- [ ] Verify app signature (advanced)

### Phishing Attack Prevention

#### Common Phishing Methods

**1. Fake website phishing**
```
Real domain: metamask.io
Phishing domain: metamask.io-wallet.com
           metamask-wallet.io
           metamask-io.com
```

**2. Fake customer service phishing**
- Pretend to be official customer service on social media
- Claim to "solve your problem"
- Induce you to provide seed phrase or private key

**3. Fake airdrop phishing**
- Claim high-value airdrops
- Induce visiting phishing websites to connect wallet
- Induce authorizing malicious contracts

#### Practical Anti-Phishing Tips

```
1. Use official bookmarks
   - Add frequently used websites to bookmarks
   - Access only through bookmarks

2. Carefully check domain
   - Check every character
   - Watch for Unicode character deception (e.g., ｍetamask.io)

3. Don't click links easily
   - Even if sent by friends
   - Even if "official" links in emails

4. Use hardware wallet confirmation
   - Confirm all transactions on hardware device
   - Screen shows real receiving address
```

### Immediate Action Checklist

When suspicious activity is discovered, **execute immediately**:

```
Step 1: Disconnect network
- Turn off WiFi and mobile data
- Prevent further attacks

Step 2: Transfer remaining assets
- Create new wallet using clean device
- Immediately transfer all remaining assets
- Use different seed phrase

Step 3: Check authorizations
- Use revoke.cash to revoke all authorizations
- Prevent further losses

Step 4: Preserve evidence

- Record transaction hashes

- Contact professional security team
```
