## How QR Code Phishing Attacks Work

### Why Are QR Codes So Dangerous?

```
Natural Weaknesses of QR Codes:
1. Illegibility - Human eye cannot directly identify content
2. Convenience - One scan and you're there, lowering user vigilance
3. Universality - Widely used for payments, logins, authorizations
4. Easy to Forge - Cost of generating QR codes is nearly zero
```

### Attack Process Diagram

```
Attacker Operations:             Victim Operations:
├─ Create malicious website      ├─ See QR code (social media/email/poster)
├─ Generate corresponding QR     ├─ Scan with phone
├─ Distribute QR code            ├─ Jump to malicious website
│   ├─ Social media              ├─ Think it's official page
│   ├─ Phishing emails           ├─ Enter private key/seed phrase
│   ├─ Fake customer service     ├─ Or sign malicious transaction
│   └─ Offline posters           └─ Assets stolen
```

## Common QR Code Phishing Scenarios

### Scenario 1: Fake Airdrop Claims

```
Attack Method:
1. Post "airdrop claim" QR codes on Twitter/Discord
2. Claim "limited time" "exclusive benefits"
3. After scanning, require wallet connection
4. Smart contract actually requests unlimited approval

Typical Case:
In January 2025, an "exclusive airdrop" QR code appeared in a popular NFT project's fan group.
After scanning, users were induced to authorize OpenSea listing permissions, resulting in NFT theft for multiple users.

Loss: Highest single address loss $120,000
```

### Scenario 2: Phishing Login Pages

```
Attack Method:
1. Send "account anomaly" notification with QR code attached
2. Claim "scan to verify identity"
3. Jump to counterfeit exchange/wallet login page
4. Induce users to enter seed phrase or private key

Identification Features:
- Urge immediate action ("must verify within 24 hours")
- Threaten account security ("will freeze if not verified")
- QR code pattern similar to official but with subtle differences
```

### Scenario 3: Malicious WiFi Connection

```
Attack Method:
1. Place "free WiFi" QR codes in public places
2. Intercept network traffic after connection
3. Steal unencrypted communication content
4. Or redirect to phishing pages

High-Risk Locations:
- Blockchain conferences/events
- Airport lounges
- Coffee shops
- Co-working spaces
```

### Scenario 4: Transaction Replacement Attack

```
Attack Method:
1. Replace QR code in legitimate transaction process
2. User scans and sends funds to scammer's address
3. Common in:
   - Offline OTC transactions
   - Merchant payments
   - Transfers between friends

Detection Difficulty: Extremely high (QR code looks completely normal)
```

## How to Identify Malicious QR Codes

### Pre-Scanning Checklist

```
✓ Source Verification
  □ Is it from official channels?
  □ Is it mentioned in official announcements?
  □ Is the sender identity trustworthy?

✓ Content Anticipation
  □ Is the QR code description reasonable?
  □ Are there urgent words like "emergency" or "limited time"?
  □ Does it match the current scenario?

✓ Visual Inspection
  □ Is the QR code pattern clear and complete?
  □ Are there any covering or modification traces?
  □ Are there official logos or identifiers?
```

### Technical Detection Methods

#### Method 1: Use QR Code Parsing Tools

```
Recommended Tools:
- QR Code Reader (Mobile App)
- ZXing Decoder Online (Web tool)
- QR Scanner (Browser extension)

Operation Steps:
1. Screenshot and save the QR code
2. Upload to parsing tool
3. View the parsed URL
4. Check if the domain is legitimate
```

#### Method 2: Security Scanning Apps

```
Recommended Apps:
- Kaspersky QR Scanner
- Norton Snap
- Tencent Mobile Manager (built-in)

Feature Highlights:
- Automatically detect malicious links
- Preview URL content
- Check domain reputation
- Warn about dangerous websites
```

#### Method 3: Manual Domain Checking

```
After parsing the URL, check:

1. Domain Similarity
   Official: binance.com
   Scam: binance-ex.com, binαnce.com

2. Protocol Type
   Safe: https://
   Dangerous: http:// (unencrypted)

3. Path Anomalies
   Normal: /login
   Suspicious: /verify-urgent-action

4. Parameter Check
   Suspicious parameters: ?token=xxx&urgent=true
```

## Real QR Code Phishing Case Studies

### Case 1: Fake Binance Customer Service

**Time**: February 2025

**Attack Process**:
1. User seeks help on Telegram
2. "Customer service" sends QR code via DM
3. Claims "scan to verify account ownership"
4. QR code points to `binance-verification.net`

**Key Detection Points**:
- Domain has extra "verification"
- Page asks for complete seed phrase (legitimate support won't ask for this)
- QR code sent by "customer service" via DM (official support doesn't send QR codes)

**Loss**: 3 users lost a total of $95,000

---

### Case 2: Conference WiFi Phishing

**Time**: March 2025 (During ETH Denver)

**Attack Process**:
1. Attacker places "Conference WiFi" sign at venue
2. Multiple attendees scan QR code to connect
3. Connected to attacker-controlled WiFi hotspot
4. Man-in-the-middle attack steals transaction data

**Key Detection Points**:
- WiFi name too generic (official conference WiFi usually has specific name)
- No password required (open networks are high-risk)
- Sign appears poorly made

**Impact**: Approximately 20 attendees were attacked to varying degrees

---

### Case 3: Merchant Payment Code Replacement

**Time**: December 2024

**Attack Process**:
1. An offline merchant uses QR code for payment
2. Attacker secretly replaces payment code when merchant isn't looking
3. Customer scans and funds go to attacker's address
4. Merchant doesn't receive payment, customer has already paid

**Detection Difficulty**:
- QR code appearance completely normal
- Neither merchant nor customer noticed
- Only discovered during reconciliation

**Loss**: Merchant lost approximately $15,000 in revenue

## Protection Strategies: Safe QR Code Usage

### Basic Protection Principles

#### 1. Three Don'ts Principle

```
Don't Scan QR Codes from Unknown Sources:
- From strangers on social media
- Attached in emails
- Shared in forums/group chats

Don't Scan Content-Suspicious QR Codes:
- Claiming "won lottery" "airdrop"
- Requiring "urgent verification"
- Promising "high returns"

Don't Scan QR Codes with Unclear Functions:
- Don't know where it will redirect
- Don't know what operation it will trigger
- Don't know what permissions it will authorize
```

#### 2. Double Verification Principle

```
After Scanning QR Code:
1. Don't operate immediately
2. Check the URL domain
3. Confirm with official channels
4. Use official App instead of web page

Example:
❌ Scan QR code → Directly enter password
✓ Scan QR code → Check URL → Manually enter official website → Login
```

### Advanced Protection Measures

#### Hardware Isolation Solution

```
Daily Phone (May be infected):
- Doesn't store large amounts of assets
- Doesn't install wallet Apps
- Only used for communication and browsing

Dedicated Device (Completely isolated):
- New phone or hardware wallet
- Only install official wallet Apps
- No social software installed
- Only used when making transactions
```

#### QR Code Whitelist

```
Build Trusted QR Code Library:
- Save official QR codes from frequently used service providers
- Compare before scanning new QR codes
- Regularly update whitelist

Example Whitelist:
□ Frequently used exchange payment codes
□ Frequently used wallet address QR codes
□ Verified merchant payment codes
```

### Enterprise/Merchant Protection

#### Payment Code Protection

```
Physical Protection:
- Use tamper-proof materials
- Place within surveillance range
- Regularly check integrity

Technical Protection:
- Use dynamic QR codes
- Set amount limits
- Enable payment notifications
- Regularly change receiving addresses
```

#### Employee Training

```
Training Content:
□ Common QR code phishing techniques
□ Methods to identify malicious QR codes
□ Handling procedures for suspicious QR codes
□ Regular security drills

Emergency Process:
1. Report suspicious QR code immediately
2. Suspend related business operations
3. Technical department verification
4. Contact police if necessary
```

## Emergency Response After Being Scammed

### Immediate Action (Within 5 Minutes)

```
Scenario 1: Connected to malicious website but haven't entered information
- Close page immediately
- Clear browser cache
- Check for abnormal authorizations
- Use Revoke.cash to revoke suspicious authorizations

Scenario 2: Already entered seed phrase/private key
- Immediately transfer assets using other secure devices
- Don't close phishing page (preserve evidence)
- Screenshot and save all information
- Report to police immediately

Scenario 3: Already signed malicious transaction
- Transaction may be in progress
- Try to cancel using Flashbots (if possible)
- Monitor stolen asset flows
- Contact exchange to freeze (if flowing to CEX)
```

### Evidence Collection

```
Evidence to Preserve:
□ QR code screenshot/photo
□ Redirected URL
□ Page screenshot
□ Transaction hash (if any)
□ Conversation records
□ Time records

Evidence Use:
- Police report materials
- Exchange appeals
- Security research sharing
- Insurance claims
```
