## 什么是Permit2？

### 背景：ERC-20授权问题

传统的ERC-20代币转账需要两步：
1. **approve()** - 授权合约使用你的代币（需要Gas）
2. **transferFrom()** - 实际转账（需要Gas）

用户需要支付两次Gas费，体验差。

### Permit2的解决方案

Permit2是Uniswap开发的授权合约，允许**离线签名授权**：
- 用户签名授权（无需Gas）
- 第三方提交交易执行（支付Gas）

```solidity
// 传统授权
function approve(address spender, uint256 amount) external returns (bool);

// Permit2授权（离线签名）
function permit(
    address owner,
    PermitSingle memory permitSingle,
    bytes calldata signature
) external;
```

### EIP-2612：原生Permit

部分代币（如USDC、DAI）支持EIP-2612标准，原生支持离线签名授权。

### 为什么危险？

## 如何识别Permit2钓鱼

### 签名前检查清单

**必须确认**：

1. **spender地址**
   - 应该是你信任的合约（如Uniswap、1inch）
   - 如果是陌生地址，**立即停止**

2. **amount字段**
   - 如果显示极大数字（如146150163733...），这是无限授权
   - 正常应该是具体金额

3. **domain.name**
   - 确认是官方Permit2合约
   - 攻击者可能使用假合约

## 防范措施

### 使用硬件钱包

硬件钱包可以显示签名内容的详细解读：
- Ledger Live 会解析Permit2参数
- 明确显示spender地址和授权额度

### 批准限额控制

使用Permit2时设置合理的限额：
- 不要使用"无限"授权
- 按需授权，用完后立即撤销
