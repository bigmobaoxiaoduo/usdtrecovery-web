## 什么是Rug Pull（跑路）？

### 定义

Rug Pull（拉地毯）是指项目方在吸引投资者投入资金后，突然卷走流动性池中的资金，导致代币价格归零的行为。

### 主要类型

| 类型 | 描述 | 占比 |
|------|------|------|
| **硬跑路** | 直接撤走流动性 | 45% |
| **软跑路** | 团队停止开发，慢慢归零 | 30% |
| **智能合约漏洞** | 预留后门盗取资金 | 20% |
| **市场操纵** | 巨鲸拉盘后抛售 | 5% |

### 跑路常见手法

#### 1. 流动性撤回（最常见）

```
操作方式：
1. 项目方提供初始流动性（如100 ETH + 代币）
2. 吸引投资者购买代币
3. 代币价格上涨，流动性增加
4. 项目方突然撤回全部流动性
5. 代币价格归零，投资者无法卖出
```

#### 2. 智能合约后门

```
常见后门：
- 隐藏的老板权限（可以无限铸造代币）
- 白名单功能（特定地址可以交易）
- 交易税修改（可以设置100%交易税）
- 黑名单功能（阻止特定地址出售）
```

#### 3. 闪电贷攻击

```
攻击流程：
1. 利用闪电贷借入大量资金
2. 操纵价格预言机
3. 从协议中套利
4. 归还闪电贷
5. 留下被掏空的协议
```

## 尽职调查（Due Diligence）清单

### 第一步：团队背景调查

#### 基本信息验证

```
必查清单：
□ 团队成员真实姓名
□ LinkedIn档案验证
□ 过往项目经历
□ GitHub技术贡献
□ 社交媒体活跃度
```

**实用工具**：
- LinkedIn：验证职业背景
- GitHub：验证技术能力
- Twitter：查看历史发言
- Wayback Machine：查看历史网页

#### 匿名团队风险评估

```
匿名 ≠ 一定是骗局，但需要更谨慎

如果团队匿名，额外检查：
□ 代码质量是否专业
□ 是否有知名审计
□ 社区是否有技术大佬背书
□ 项目运行时间长短
□ 资金是否安全（多重签名、时间锁）
```

### 第二步：智能合约审计

#### 审计报告检查

```
知名审计公司：
- CertiK
- OpenZeppelin
- Trail of Bits
- ConsenSys Diligence
- PeckShield

检查要点：
□ 是否是知名审计公司
□ 审计日期是否近期
□ 发现的严重问题是否修复
□ 是否有未修复的高危漏洞
```

#### 自主代码检查

```
使用工具：
- Etherscan：查看合约代码
- DeFi Safety：安全评分
- RugDoc：风险检查

检查清单：
□ 合约是否已验证
□ 是否有owner权限
□ 是否有mint功能
□ 是否有黑名单功能
□ 交易税是否可以修改
```

### 第三步：代币经济模型分析

#### 持币分布检查

```
使用工具：
- Etherscan：Token Holders
- DeBank：代币分析
- Bubblemaps：可视化持币分布

危险信号：
□ 前10地址持有>50%（除非是交易所）
□ 团队钱包无锁仓
□ 大量代币集中在合约地址
□ 近期有大量代币转移
```

#### 流动性分析

```
关键指标：
1. 流动性/市值比
   - 健康：>20%
   - 警告：10-20%
   - 危险：<10%

2. 流动性锁定
   - 必须检查是否锁定
   - 锁定期至少6-12个月
   - 使用Uncx或Team Finance等可靠平台

3. LP代币归属
   - 谁持有LP代币
   - 是否可以随时撤流动性
```

### 第四步：社区和营销评估

#### 社区健康度

```
检查渠道：
□ Discord活跃度
□ Telegram群组质量
□ Twitter真实互动
□ Reddit讨论氛围

红旗信号：
- 机器人充斥
- 不允许提出质疑
- 只有价格讨论，无技术讨论
- 管理员经常禁言用户
```

#### 营销策略评估

```
健康项目的营销：
✓ 技术导向的内容
✓ 透明的开发进展
✓ 社区驱动的增长
✓ 长期的路线图

危险信号：
✗ 过度承诺收益
✗ 名人站台（付费推广）
✗ 拉人头奖励
✗ "错过就晚了"的紧迫感
```

**时间**：2021年10月

**背景**：
- 蹭Netflix《鱿鱼游戏》热度
- 代币价格几天内上涨230,000%
- 声称可以"玩游戏赚取代币"

**红旗信号**：
- ❌ 网站有大量拼写错误
- ❌ 无法出售代币（合约限制）
- ❌ 团队完全匿名
- ❌ 社交媒体突然爆发（机器人）

**结局**：
- 项目方卷走约$3.4 million
- 代币归零
- 无数投资者血本无归

**教训**：
> 如果无法出售代币，那就是蜜罐！

## 风险管理策略

### 投资组合配置

```
DeFi投资原则：
- 只投入你能承受损失的资金
- 单个项目不超过总仓位的5%
- 新项目投资不超过总仓位的1%

建议配置：
- 70%：主流币种（BTC、ETH）
- 20%：主流DeFi协议（Aave、Uniswap等）
- 10%：高风险新项目（进一步分散）
```

### 止盈止损策略

```
设定规则：
1. 止损
   - 单个项目亏损>50%：重新评估
   - 亏损>80%：考虑止损

2. 止盈
   - 盈利>2倍：收回本金
   - 盈利>5倍：取出一半利润
   - 剩余部分长期持有

3. 再平衡
   - 每月评估一次
   - 获利项目适当减仓
   - 补充到更稳健的资产
```

### 持续监控清单

```
投资后的持续监控：

每周检查：
□ 价格异动（>30%波动）
□ 流动性变化
□ 社交媒体动态

每月检查：
□ GitHub代码更新
□ 团队动态
□ 竞品发展
□ 监管动态

紧急退出信号：
🚨 流动性突然撤走
🚨 团队核心成员离职
🚨 合约权限变更
🚨 大量代币转移
```

## 如果你已经投资了可疑项目

### 紧急评估

```
立即检查：
1. 能否出售代币？
   - 在小额测试出售
   - 如果无法出售，可能是蜜罐

2. 流动性状况
   - 检查流动性是否还在
   - 检查LP是否锁定

3. 团队动态
   - 社交媒体是否活跃
   - 是否有异常言论

4. 社区氛围
   - 是否允许质疑
   - 是否有人反映问题
```

### 退出策略

```
情况一：可以正常交易
- 如果已经盈利：考虑分批止盈
- 如果亏损：评估是否值得继续持有

情况二：怀疑即将跑路
- 立即出售（即使亏损）
- 不要贪心等待回本
- 及时止损保存本金

情况三：已经跑路
- 保存所有证据
- 记录交易哈希
- 联系其他受害者
- 报警并寻求专业帮助
```
